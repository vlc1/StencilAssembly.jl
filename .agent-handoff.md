## Key Decisions

- **Row/col representation**: `NTuple{N, AbstractUnitRange{Int}}` on a shared
  integer mesh. Rectangular sub-blocks; unequal lengths and shifted ranges
  supported. Arbitrary boolean masks are out of scope.
- **Coefficient indexing**: column-anchored, `coefs[k][c]` at the column mesh
  position. Caller's responsibility for axes covering `col` (wrap with
  `OffsetArray` if needed).
- **Strict-descending offsets** is the CSC-sortedness invariant; enforced at
  the `LinearStencil` inner constructor via `issorted(offsets; lt = >=)`.
- **`D ≤ N`** is enforced at the inner constructor (stencil dim must fit
  within coef-array dims).
- **Kernel shape**: `K` outer loops, one per offset; counting-sort-CSC with
  `colptr` mutated in place as slot tracker, restored via shift-right at
  the end. Single-threaded; mid-fill `colptr` is briefly inconsistent.
- **API surface**: `LinearStencil`, `assemble`, `update!`, `build` (exported);
  `_pattern!`, `_fill!` internal.
- **For the upcoming N-D milestone**: tuple-length dispatch (`Base.front` /
  `last`), separate methods for `NTuple{1, …}` base and `NTuple{Nd, …}`
  recursive. No `Ref`s — accumulators returned as `Tuple`s. K-outer-loops
  shape generalises (each `k` becomes a sub-stencil with a rectangular
  c-range).
- **Symbolic/numeric split (`StencilPattern` vs `LinearStencil`)** considered
  and **deferred** to the N-D design pass.

## Session 2026-05-18 (Continued): N-D Recursion & colptr/row_offset Fix

### Work Completed This Session
1. **Fixed colptr construction in N-D case**
   - Threaded `col_idx` (global column index) through `_pattern_nd_recursive`
   - Implemented direct row index computation in base cases using `row_offset`
   - Computed `row_offset = (outer_col_idx - 1) * length(row[1])` for global row indexing
   - Added `outer_col_idx` parameter to track which column in outer dimensions we're at

2. **Fixed fill phase to match pattern phase**
   - Updated `_fill_nd_recursive` with same `col_idx` and `outer_col_idx` threading
   - Implemented base case fill logic that indexes coefs properly: `coefs[k][c1, outer_col_idx]`
   - Handled both D==1 stencil and D>N_dims intersection cases in fill

3. **Fixed pattern and fill for D > N_dims case**
   - Both cases now iterate over all offsets and emit entries for valid rows
   - Pattern case: emit row indices for each valid offset
   - Fill case: emit coefficient values using proper N-D indexing

4. **Test Results**
   - ✓ 2-D D=1: PASSING (stencil along dimension 1 works correctly)
   - ✗ 2-D D=2: FAILING (pattern emits correct entries but missing three-phase logic at dimension 2)
   - ✓ 1-D tests: still passing (45 assertions)

### Critical Issue Remaining & Solution
**Problem**: D=2 case not applying three-phase logic
- Current code at lines ~449: simplified stencil-sweep case just does `for i_last in 1:count` with single recursion
- This doesn't compute variable `active(c_last)` from three-phase logic
- Result: col 4 gets only 1 entry [4] instead of 2 entries [1, 4]

**Key Insight**: For D==N_dims (stencil at outermost dimension), the recursion must:
1. Apply three-phase logic to determine how many rows each column gets
2. Recurse ONCE per column, passing information about which rows are valid
3. Lower dimensions emit those rows, not compute their own row counts

**Fix Strategy**: Two options:
A. **(Simpler)** Restore the three-phase loop (left ramp, interior, right ramp) but DON'T loop `for _ in 1:active`.
   Instead, just recurse once per column with proper active count information.
B. **(Cleaner)** Thread the active count or offset information to base case so it can compute rows correctly.

**Recommended approach**: Option A - restore three-phase logic at lines ~449-480, but simplify the recursion loop.
In stencil-sweep case: iterate over c_last values (not active counts), compute active(c_last), then recurse once
with that information somehow threaded through.

- All 1-D tests still passing (31 assertions)

### Critical Issue Identified
- **colptr construction in N-D case**: Current recursive structure doesn't
  properly map local column indices (from `_pattern!` base case) to global
  indices. The base case calls `_pattern!(rowval, colptr, ..., row[1], col[1])`
  which fills `colptr` with indices 1 to n relative to `col[1]`, but the
  N-D recursion needs global indices across all outer dimensions.
  
  **Impact**: 2-D tests fail with misaligned colptr/rowval.
  **Fix**: Thread global column index through recursion; emit rows manually
  in base case rather than calling `_pattern!` directly.
